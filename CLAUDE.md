# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

GitHub **profile README** repo (`MaxCarroll0/MaxCarroll0`). The README's three tables (Totals,
Top Languages, Recently Used) are generated by Lua scripts under `scripts/` and refreshed daily by
`.github/workflows/cards.yml`. There is no application code beyond the generator.

## Dev environment

`flake.nix` provides a devShell (lua5.4, tokei, git, gh, yq-go, lua-language-server, stylua, taplo) plus
a runnable package. `.envrc` is `use flake`, so `direnv allow` (or `nix develop`) puts the toolchain on
PATH. The Lua LSP config is baked into the flake: the devShell ships `lua-language-server` wrapped with
`--configpath` pointing at a Nix-generated luarc, so there is no committed `.luarc.json`. Format Lua with
`stylua scripts/` (config `stylua.toml`).

## Generating stats

```
GH_TOKEN=$(gh auth token) nix run .                          # full run via the flake, writes README
GH_TOKEN=$(gh auth token) lua5.4 scripts/stats.lua           # same, from inside the devShell
lua5.4 scripts/stats.lua --dry-run                           # print tables, leave README alone
lua5.4 scripts/stats.lua --only=hazelgrove/hazel --dry-run   # one repo/group, fast iteration
```

Run from the repo root (the script reads `config.toml` and writes `README.md` relative to the cwd).
`--only=SUBSTR` filters owned+external by `owner/repo` substring (`--only=owned` limits to owned). The
script rewrites the regions between marker pairs in `README.md` (`<!-- TOTALS:START/END -->`,
`<!-- TOP-LANGS:... -->`, `<!-- RECENT-LANGS:... -->`); a missing marker pair is a hard error.

## Configuration

`config.toml` (repo root): identities (emails), owned-repo filters, the `[[external]]` repo+ref list,
`recent.days`, language `hide`/`merge`, `[groups]` (order, per_group, fold/overrides), output markers.
Loaded by `lib/conf.lua`, which shells out to `yq` (falling back to `tomljson`/`taplo`) to convert TOML
to JSON and decodes with `lib/json.lua`; there is no hand-rolled TOML parser.

## How counting works (the non-obvious parts)

Two metrics use **different signals on purpose**, because owned repos are not blamed:

- **Top Languages (all-time):** owned repos use `tokei` `code+comments` of the final tree (wholesale,
  you own them); external repos count lines attributed to your `emails` via `git blame -w
  --line-porcelain`, non-blank lines only, **deduped by originating commit** so the three shared-history
  `hazel` branches do not double-count.
- **Recently Used (last N days):** *all* repos count lines you **added** in commits within the window
  (`git log --author --since --numstat`), deduped by SHA. This is churn, so it can exceed the all-time
  counts; the column is "Added". The README's "last 90 days" text mirrors `recent.days`.

Languages are classified into **Programming / Prose / Configs-Data** and coloured by GitHub Linguist.
`lib/linguist.lua` (a `{lang = {t=type, c=color}}` table) is GENERATED from Linguist's `languages.yml` by
`scripts/gen-linguist.lua`. `lib/langgroups.lua` folds the four Linguist types into three groups (markup
goes to Prose by default; override per type with `[groups].fold` or per language with `[groups.overrides]`,
e.g. Nix to Configs / Data) and exposes `color(lang)` (merged names like "OCaml/Reason" resolve to the base).

The two language metrics render as three side-by-side classes (one HTML `<td>` each). Per class, `lib/svg.lua`
builds a stacked proportion bar (above the table) and, for each row, an outlined-box bar filled left-to-right
to the within-group share, both coloured by the language's Linguist colour so a row's bar and its stacked
segment match. `lib/render.lua` emits the HTML plus the SVGs (written under `assets/`, re-derived and pruned
each run); `lib/mdtable.lua` still aligns the inner Markdown tables and the Totals table.

Per-repo work is `pcall`-isolated. External repos are full-cloned (blame needs history); multiple refs of
one repo share a clone via `git worktree`. Owned repos use `--shallow-since`. All clones disable git-lfs
filters so tokei is not blocked on (and does not fetch) LFS binaries.

## Module map (`scripts/`)

`stats.lua` orchestrates. `lib/`: `conf` (TOML to JSON), `json` (decoder), `github` (owned discovery via
`gh api --paginate --slurp`), `repos` (clone/worktree), `tokei` (run + parse: per-file stats live in
`reports[].name`/`.stats`, skip the `Total` key), `blame` (attribution + dedup), `churn` (recent),
`langgroups` plus generated `linguist` (classification + colour), `aggregate` (group/sort/cap,
within-group %), `svg` (per-row + stacked bar SVGs), `render` (3-column HTML tables + the SVG files +
totals), `mdtable` (align Markdown), `inject` (marker replacement), `util`.
`require` paths assume the run starts at the repo root.

## CI

`cards.yml` installs Nix and runs `nix run .` (the flake's `readme-stats` package, the same toolchain as
the devShell) with `GH_TOKEN=secrets.GH_PAT`, then commits the README and the `assets/` SVGs. `GH_PAT` needs `repo` scope for
private owned repos and the private `Talking-Music`; an inaccessible repo is skipped, not fatal.
